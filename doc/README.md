## Documentation Guidelines

### Writing Style
- Clear and concise
- Use active voice
- Include examples
- Add code snippets
- Include screenshots

### Format
- Markdown for all docs
- Diagrams in SVG/PNG
- Code in TypeScript/Vue
- Screenshots in PNG/WebP

### Maintenance
- Keep docs up to date
- Review monthly
- Update with features
- Version control
- Link checking
